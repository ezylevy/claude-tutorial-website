Claude Code (CLI)

למפתחים

כלי שורת פקודה (Command Line Interface) שעובד בטרמינל. מיועד בעיקר למפתחים ואנשי טכנולוגיה.

מתאים ל:
  • כתיבת קוד ופיתוח
  • דיבאג ותיקון באגים
  • ניהול פרויקטים טכניים
  • עבודה עם Git
  • הרצת פקודות מערכת
  • אוטומציה של תהליכי פיתוח
איך עובדים:
  • פותחים טרמינל (PowerShell)
  • מקלידים פקודות טקסט
  • Claude קורא/כותב קבצים
  • מריץ פקודות במערכת
claude "צור קומפוננטת React לטופס התחברות"
🖥️ Windows 10/11 או Mac/Linux
🎨

פתיחת PowerShell

PowerShell הוא הטרמינל של Windows. יש כמה דרכים לפתוח אותו:

אפשרות 1: חיפוש רגיל

לחצו על Start והקלידו "PowerShell", אז Enter.

זו הדרך הכי נוחה לעבודה עם Claude Code:

  • Windows 11: קליק ימני על תיקייה → "Open in Terminal"
  • Windows 10: החזיקו Shift + קליק ימני על תיקייה → "Open PowerShell window here"

התקנת Claude Code

עכשיו מגיע החלק המרכזי - התקנה בפקודה אחת!

מומלץ

אפשרות 1: סקריפט PowerShell

העתיקו והדביקו את הפקודה הזו ב-PowerShell:

irm https://claude.ai/install.ps1 | iex

לחצו Enter והמתינו. זה הכל!

מה קורה פה? irm מוריד את הסקריפט, ו-iex מריץ אותו. הכל אוטומטי!

אפשרות 2: WinGet

אם אתם מעדיפים את מנהל החבילות של Windows:

winget install Anthropic.ClaudeCode

וידוא התקנה

פתחו PowerShell חדש והקלידו:

claude --version

אם מופיע מספר גרסה - ההתקנה הצליחה!

⚠️
שגיאת execution policy?

הריצו קודם:

Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

אימון Claude עם קובץ CLAUDE.md

קובץ CLAUDE.md הוא הדרך הכי חשובה לאמן את Claude להבין את הפרויקט שלכם ולעבוד בצורה מדויקת יותר.

למה זה חשוב?

  • Claude קורא את הקובץ אוטומטית בכל פעם שמפעילים אותו בתיקייה
  • הוא מבין את ההקשר, הסגנון והדרישות שלכם
  • חוסך לכם חזרות על הנחיות
  • מבטיח תוצאות עקביות

איך ליצור קובץ CLAUDE.md?

צרו קובץ בשם CLAUDE.md בשורש (root) של תיקיית הפרויקט:

# שם הפרויקט

## תיאור
תיאור קצר של הפרויקט ומטרתו.

## טכנולוגיות
- React 18
- TypeScript
- Tailwind CSS

## מבנה התיקיות
- /src - קוד המקור
- /components - קומפוננטות React
- /api - קריאות API

## כללים והנחיות
- כתוב קוד נקי ומתועד
- השתמש בעברית בתגובות ובטקסטים
- עקוב אחרי קונבנציות הפרויקט
- אל תשנה קבצי קונפיגורציה בלי לשאול

## סגנון קוד
- השתמש בפונקציות קטנות וממוקדות
- הוסף תיעוד לפונקציות מורכבות
- עקוב אחרי ESLint rules

## דברים שחשוב לדעת
- הפרויקט משתמש ב-API חיצוני בכתובת...
- יש לבדוק תאימות ל-IE11
- הנתונים מגיעים בפורמט...

טיפים לכתיבת CLAUDE.md אפקטיבי:

✓ היו ספציפיים

במקום "כתוב קוד טוב" כתבו "השתמש ב-TypeScript strict mode"

✓ תנו דוגמאות

הוסיפו דוגמאות לסגנון הקוד המועדף

✓ הגדירו גבולות

ציינו מה Claude לא צריך לעשות

✓ עדכנו תמיד

# Skill: create-component

## תיאור
יוצר קומפוננטת React חדשה עם כל הקבצים הנדרשים.

## פרמטרים
- name: שם הקומפוננטה (חובה)
- path: נתיב יחסי (ברירת מחדל: src/components)

## שלבים לביצוע

1. **צור תיקייה** בשם `{name}` בנתיב `{path}/{name}/`

2. **צור קובץ index.tsx** עם הקוד:
```tsx
import React from 'react';
import styles from './{name}.module.css';
import { {name}Props } from './types';

export const {name}: React.FC<{name}Props> = ({ children }) => {
  return (
    

      {children}
    

  );
};

export default {name};
```

3. **צור קובץ types.ts**:
```ts
export interface {name}Props {
  children?: React.ReactNode;
}
```

4. **צור קובץ {name}.module.css**:
```css
.container {
  /* styles here */
}
```

5. **צור קובץ {name}.test.tsx**:
```tsx
import { render, screen } from '@testing-library/react';
import { {name} } from './index';

describe('{name}', () => {
  it('renders children', () => {
    render(<{name}>Test);
    expect(screen.getByText('Test')).toBeInTheDocument();
  });
});
```

6. **עדכן exports** בקובץ src/components/index.ts

## פלט
הודעה על יצירה מוצלחת עם רשימת הקבצים שנוצרו.
